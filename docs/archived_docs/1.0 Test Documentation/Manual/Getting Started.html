Shard Entries:

To setup Shard entries, you will need to find the IP of your network or internet adapter, or use 127.0.0.1 for a same machine entry client. Shard entries consist of three lines, a name, an IP address to connect to/through and a port. Shard entries can be given any name, but the IP address and port must be a valid UOX server.

Note: It is possible to have several shard entries set up in your UOX3.cfg file.
          These entries must have the correct IP for people entering via that
          Shard on the Shard choice screen. Example, is you have a network
          address people connect on, a internet address people connect on and a
          server administrator playing on the same system the server is on, then
          you will need three shard entries set up with the different addresses as
          shown below (from UOX3.cfg):

          UOX3 World (Internet Clients only)
          256.351.2.9
          2593
          UOX3 World (Network Clients only)
          192.168.0.6
          2593
          UOX3 World (Local Host only)
          127.0.0.1
          2593

World Entry Points:

If you are using a custom world map (ie, a completely different world from OSI's, or one created in Green Acres), you may need to change the world entry points for new players. Each of the named entries corresponds to the towns on the "town entry" screen that players are presented with as the last choice of character creation. These value are XYZ coordinates and can be found in-game by using the 'where command  while using a player with gm privileges (normally any player on account 0)

 

These values can be changed at any time without affecting any other components of the Shard. This concludes the setup of uox3.ini.

 

(ACCOUNTS.ADM) Open the ACCOUNTS.ADM file found in the UOX folder.At the top you will see   SECTION ACCOUNT 0,in here you will put the Username and Password of the Char who will have the GM stats. You can use any name that you want in there. And you can add as many accounts as you see fit. Just add them exactly as the first two accounts are setup. And remember the two empty lines before the EOF at the END of the accounts file, no matter how many accounts you have.
Also note that the first account in the list is reserved for the  server operator and has some special privileges, like having GM rights on every fresh created character.

2) Set up your Network: (If needed) To play UOX3 you will need at least one IP adress assigned to your machine. This is required because of a client bug which will crash the client after playing the intro movie if there are no IP adresses available. Because its in the Client, we can not do anything about it. You can either:

a) Connect to your ISP which automatically assigns an IP to you, then press START and RUN and write in "winipcfg". Now you should see your internet IP in the window that pops up. Another way to see it is to do /dns (without the < >) while on a IRC server.

b) Set yourself a static IP adress. Go to the Win95 control panel and then select Network Settings. Then single click on the TCP/IP entry in the enabled network componets box, then click the properties button. Now click the radio button labeled "specify an IP address". Now, you are able to fill in the two boxes below it. In the first one, the IP address box, you need to enter an IP address of your choice. ("1.1.1.1" works, but any other number will do.) Then, set the subnet mask to 255.255.255.0. Click "OK" and windows will ask you to restart your computer, click "OK"

*IMPORTANT* Remember, you need to re-enable the upper radio button before you can connect to your ISP the next time.

c) Type "ping localhost" on the command line of an open ms-dos prompt window. First line should be like: "Pinging zeus [127.0.0.1] with 32 bytes of data:" The name before the [127.0.0.1] will be different, as it is your computers name. Now edit the file named "hosts" in your windows directory and add a line like "127.0.0.1 zeus" below the one that reads "127.0.0.1 localhost". Again, remember to replace "zeus"  with your computers name. Once you reboot Windows, the client should work without crashing.

If you are on a LAN, you probably already have a permanent IP adress  assigned to your network card and can skip this step. Also remember that if you choose possibility b) or c) people from the internet can  not connect to your server. Choose a) if you want multiplayer.

3) Setup the program:  Copy your login.cfg to login.old as a backup.  Unzip all UOX files into one directory.

4) Run the server:  Run UOX3.EXE  If all's fine, it will print "UOX3: Startup complete." on the console.

5) Change login information:  Now edit your LOGIN.CFG (you backed it up, didn't you?) to the shown  IP adress. So if it were the one displayed under 3), your LOGIN.CFG  would read:

;Loginservers for Ultima Online
;Do not edit this file or patching will fail!  Always save a backup.
LoginServer=1.2.3.4,2593
LoginServer=1.2.3.4,2593
LoginServer=1.2.3.4,2593
LoginServer=1.2.3.4,2593

Use IP 127.0.0.1 for a single player game by yourself and for a multi-player game,You will need to setup your internet IP address. You will also need to give that IP adress to everyone trying to log  in to your system. (A good program to keep contact to your friends for that purpose is called ICQ. You can download it at  www.mirabilis.com) When editing login.cfg, don't forget the port number. In the above example, 2593 is the port.

6) Start Ignition which you'll find in a subfolder of uox. It will start the UO client and bypass the encryption so you can log into your UOX3 shard. The current Ignition version will only work with UO Clients 2.0.0 -> 2.0.0e.
