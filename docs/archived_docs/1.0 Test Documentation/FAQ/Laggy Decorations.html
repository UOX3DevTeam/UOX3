What can be done to stop decorations from causing lots of lag?

Well, fortunately, there is a program which was designed for situations like this. It goes by the name of Worldforge, and is available from http://blah.blah.blah/blah. Worldforge serves two functions that help solve this problem:

1. Freezing:

This function allows you to add your decorations to your statics0.mul by "freezing" them. This would make your new decorations and buildings static items, so not only is your lag cut down, your can also use circle of transparency on them. Freezing will save type, position, and hue, but any other values will be lost. Therefore you cant use this on items that need to be triggered.

To freeze an item, just set its type to 255 ('settype 255) in Ultima Online and then hit the freeze button in Worldforge.

2. Melting:

This function allows you to "unfreeze" 8x8 blocks of items that you have previously frozen, in order to delete them or edit them. You can then "refreeze" those items once you are done editing.

To melt a block of items, find the coordinates in Ultima Online by using the where command ('where) and then divide both x and y coordinates by 8. Put these coordinates into the corresponding Worldforge boxes and then hit melt. To refreeze the items, just hit the freeze button in Worldforge.
