UOX3 NPC Scripting Guide
Version 1.0

This is a guide to the current NPC scripting system. It is designed to show you how to script your own npcs, and how they work in game. Currently it shows what a npc script is made of and explains the use of the npcrect command. Below is the entry for a ratman mage:

Heres a line by line explanation of that script, and whether the line is necessary in a npc script section:

Line #:Necessary:Contents:
1Yes This line sets the npc number. This number must be unique within your npc.scp or it will cause some unpredicatable bugs.
2Yes This line opens the npc section
3Yes This line uses the NAME command. This is the name that will be displayed in game. You may also use namelists in this line. (eg NAMELIST 500)
4Yes The id of the npc. Make sure this is not a item id. You can retrieve ids from InsideUO.
5No This line contains the skin colour of the npc. It changes the hue of the monster id to this colour (colours are listed in Xuris UOX guide)
6No The direction command sets the direction the npc is facing when they are first spawned
7? This line states that the npc has a backpack
8No The pack item command adds an item into the monsters pack. If you want players to be able to steal items off of npcs then you need to add them via packitem lines. You may have as many packitem lines as you want.
9No Skill line. This is the name of a skill followed by 1 or 2 amounts. If you have 2 amounts uox will spawn the npc with a skill between the 2 stated amounts. You can give a npc as many skills as you want, but i would suggest you dont give them too many more than the player characters can have. The values are in tenths of a UO skill point, so 500 would be a skill of 50.
10? Spell attack value. Most likely needed for mage npcs.
11? Delay in real life seconds between casts. Most likely needed for mage npcs.
12No This line sets the loot generated when the npc is killed. You must use a valid loot list number. You may have as many loot entries as you wish.
13No Same as above.
14No Same as above.
15No This line sets the gold the npc carries. You can have 1 one number or 2 if you wish the gold to be variable.
16No Sets the strength, if you do not include this line it will default to 10.
17No Sets the dexterity, if you do not include this line it will default to 10.
18No Sets the intelligence, if you do not include this line it will default to 10.
19No This line sets the karma of the monster. Note this does not add or subtract this amount to your karma if you kill it. If you dont include this line it defaults to 0.
20No This sets the fame of the monster. Note that this isnt the fame you get if you kill it. You can only gain fame when you kill npcs with more fame than you. If you dont include this line it defaults to 0.
21No More skill lines. These are at a fixed level.
22No Same as above.
23No Same as above.
24No Same as above.
25No This is a comment line. It will be ignored by UOX. Comments in the current script system are in the following format: "// text"
26? This is the range of combat damage a npc can do. You may also state a single value.
27? This is the in-game armour value of the npc. Same as the players ar ratings.
28? This is the type of wander mode. Refer to the gm commands or the sections below.
29? This is used in conjunction with the NPCWANDER line and refers to where the npc spawns/is allowed to walk. For more information see the gm commands or the sections below.
30? Same as above.
31? Same as above.
32? Same as above.
33No Sets a special npc ai if needed. The types of npc ai that can currently be chosen are:
34No This is the least amount of taming skill you would need to tame this npc.
35Yes This line closes the npc section

Thats the entirety of a npc script section. There may be one or two other commands you can add to the script, and these will be added as they are found.


In the NPC script files, the meaning of the values of FX1, FX2, FY1, FY2 and FZ1 have changed.

Because a Scriptwriter doesn't know where the NPC will be placed, a valid set of 'true' coordinates cannot be given for a bounding rectangle or circle. Because of this, the values of the "F" coordinate values will not be interpreted as RELATIVE coordinates - coordinates based on where the NPC was created.

With the new system, if we set FX1=-4, this means that the NPC can go as far as 4 squares left (negative on the x-axis) from the starting point. Also, FY1=-4 gives us 4 squares up. With FX2=4 and FY2=4, our NPC will be able to walk in this area:

This means that the NPC is ALLOWED outside the shop, but there's no way for the NPC to get there. Even (-6,-6) (6,6) would be okay; they could wander around the outside of the building.

I don't really know what the size of shops usually are, and of course they can change. Without changing the NPCs wander numbers by hand, there's really no perfect way (unless we had the NPC understand that it was INDOORS, which isn't going to happen soon).

If the range is too small, then the vendor just won't walk to the edges of the shop: it's quite possible no one would ever notice that. Making the range a little too big shouldn't be bad, either; the vendor will always "be around".

So, for vendors, something like

FX1=-6
FY1=-6
FX2=6
FY2=6
FZ1=-1

and for monsters, perhaps

FX1=-12
FY1=-12
FX2=12
FY2=12
FZ1=-1

But it's up to you! Some monsters might roam farther, some not at all... of course, what this means is a pile of modifications to npc.scp. But you had nothing else to do, did you?
